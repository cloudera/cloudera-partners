* ✅ Base Cluster Deployed with major services.

== Prerequisites

Before you begin, ensure the following are installed and configured on your system:

* ✅ **AWS CLI** installed and configured (via AWS SSO or static Access Key/Secret).
* ✅ **Docker** installed and running.
* ✅ **License file** named `license.txt` present in the cloned automation directory.
* ✅ **SSH private key** available at `~/.ssh/id_rsa`.
* ✅ You are logged into AWS CLI (run `aws sso login` if using SSO, or `aws configure` for static credentials).

== Automated Deployment using Wrapper Script

To simplify the entire deployment or teardown process, use the provided wrapper script `run_onprem_deploy.sh`. This script automates virtualenv setup, credential management (with optional SSO), environment variable setup, and playbook execution.

You can deploy or teardown the PVC stack using either the automated wrapper script or the manual Ansible Navigator commands.

=== Download the wrapper script, make it executable, and run it with your desired options.

[source,sh]
----
curl -O https://raw.githubusercontent.com/cloudera/cloudera-partners/goespvc/ClouderaSetup/OnPremise/Automation/run_onprem_deploy.sh
chmod +x run_onprem_deploy.sh
----

==== Provision

**To provision using AWS SSO:**

[source,sh]
----
./run_onprem_deploy.sh deploy --sso --region <aws-region> --prefix <deployment-prefix> --owner <your-email> --password '<secure-password>'
----

**To provision using static AWS credentials:**

[source,sh]
----
./run_onprem_deploy.sh deploy --region <aws-region> --prefix <deployment-prefix> --owner <your-email> --password '<secure-password>'
----

**Example:**

[source,sh]
----
./run_onprem_deploy.sh deploy --region us-west-2 --prefix demo-pvc --owner you@example.com --password 'Str0ngP@ssword!' --sso
----

==== Destroy (Teardown)

**To teardown the deployment using AWS SSO:**

[source,sh]
----
./run_onprem_deploy.sh --sso --region <aws-region> --prefix <deployment-prefix> --owner <your-email> --password '<secure-password>'
----

**Example:**

[source,sh]
----
./run_onprem_deploy.sh --region us-west-2 --prefix demo-pvc --owner you@example.com --password 'Str0ngP@ssword!' --sso
----

**To teardown the deployment using static AWS credentials:**

[source,sh]
----
./run_onprem_deploy.sh --region <aws-region> --prefix <deployment-prefix> --owner <your-email> --password '<secure-password>'
----

==== **Arguments explained:**

* `provision` or `teardown` — action to perform.
* `--sso` — optional, use AWS SSO for authentication.
* `--region` — AWS region code (e.g., us-west-2).
* `--prefix` — unique deployment name prefix.
* `--owner` — your email address.
* `--password` — common secure password meeting complexity requirements.


NOTE: The `password` or `common_password` is reused across multiple components (e.g. FreeIPA, Ranger, Cloudera Manager UI). It must meet the following complexity requirements:

* Minimum 8 characters
* At least one uppercase letter
* At least one lowercase letter
* At least one number
* At least one special character

== Manual execution of commands after cloning the repository (Manual Method)

=== Clone the Repository

[source,sh]
----
git clone https://github.infra.cloudera.com/GOES/pvc-community-edition-aws.git
cd pvc-community-edition-aws
----

=== Set Up Python Virtual Environment

[source,sh]
----
pip3 install virtualenv
python3 -m venv ~/cdp-navigator
source ~/cdp-navigator/bin/activate
----

=== Install Required Python Packages

[source,sh]
----
pip3 install ansible-core ansible-navigator virtualenv
----

=== Configure AWS Credentials

IMPORTANT: If you are using AWS SSO, export any existing static credentials first:

[source,sh]
----
unset AWS_ACCESS_KEY_ID AWS_SECRET_ACCESS_KEY AWS_CREDENTIAL_EXPIRATION AWS_SESSION_TOKEN
aws sso login
eval $(aws configure export-credentials --format env)
----

TIP: You can also use standard Access Key and Secret if not using SSO.

=== Prepare Required Files

==== License File

Place your CDP license file in the current directory and name it:

[source,sh]
----
license.txt
----

==== SSH Private Key

Ensure your SSH private key is available at the following path:

[source,sh]
----
~/.ssh/id_rsa
----

If the file does not exist, generate a new one:

[source,sh]
----
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa
----

=== Set Environment Variables

[source,sh]
----
export SSH_PRIVATE_KEY_FILE=~/.ssh/id_rsa
export CDP_LICENSE_FILE=license.txt
----

=== Prepare Configuration File

Create a working copy of the template:

[source,sh]
----
cp config-template.yml config.yml
----

Edit `config.yml` and update the following fields:

[source,yaml]
----
name_prefix:       <unique-deployment-name>
infra_region:      <aws-region-code>
common_password:   <secure-password>
owner:             <your-email@cloudera.com>
----

=== Deploy the CDP PVC Stack

To deploy the full stack (infrastructure + services + Ozone cluster), run:

[source,sh]
----
ansible-navigator run infrastructure_setup.yml services_setup.yml cm_setup.yml ozone-cluster.yml -e @config.yml
----

=== Teardown the CDP PVC Stack

To clean up the environment when no longer needed:

[source,sh]
----
ansible-navigator run infrastructure_teardown.yml -e @config.yml
----

=== Access the Deployment Summary and UI

Once deployment is complete, you can review the deployment summary and access the available UI endpoints:

=== View Summary Page

You can either:

* Open the generated `<NAME_PREFIX>-summary.html` file from the project directory, or
* Visit the hosted URL (Listed in the webpage `<NAME_PREFIX>-summary.html`) :

[source,sh]
----
https://REVERSE_PROXY_PUBLIC_IP.pvc.cloudera-labs.com
----

This summary page provides:

* All UI endpoint URLs (Cloudera Manager, FreeIPA, pgAdmin, Knox, etc.)
* Hostnames and their assigned Ansible groups
* Service status snapshots

=== Quick Command to Open in Browser

Use this command to open the summary file directly in your default web browser:

[source,sh]
----
python -m webbrowser file://${PWD}/<NAME_PREFIX>-summary.html
----

=== What Next?

After the stack is provisioned:

* You can continue further configuration in **Cloudera Manager UI**
* Use **Knox Gateway** to interact with cluster endpoints securely
* Access **pgAdmin**, **FreeIPA**, and other UIs to explore more capabilities

== Overview
This guide provides step-by-step instructions to demonstrate the capabilities of Cloudera Data In Motion (DIM) Kubernetes Operators, including CFM Operator (for NiFi and NiFi Registry) and Streaming Operators (CSA for Flink and CSM for Kafka). It covers environment preparation, configuration, and running sample data flows and streaming jobs. The guide is intended for users who want to quickly set up and showcase data flow and streaming use cases on Kubernetes using Cloudera's operators.

== CFM-Operator

=== Preparation

Before starting the demo, ensure you have the following prerequisites:

. `CFM Operator` with `NiFi` and `NiFi Registry` installed in your Kubernetes cluster. If not installed, follow the instructions in the link:./README.adoc[`**Installation Guide**`] to set it up.

. You're able to access the `NiFi` and `NiFI Registry` web UIs. Follow the steps `4.1.` and `4.2.` in the link:./README.adoc[`**Installation Guide**`] to access UI.

=== Demo Steps
. Confirguration on `NiFi Registry` UI

.. Login to the `NiFI Registry` UI.
.. Create a new bucket name it (for example `demo-bucket`).
... Click on the wrench`🔧` icon on the top right corner to open settings.
+
image::../images/settings.png[settings]

... Click on New bucket and name it `demo-bucket`.
+
image::../images/Bucket_creation.png[bucket_creation]

.. Click on `✏️` icon to edit the bucket. Click on `NEW POLICY`, select `nifiadmin` user and grant `All` permissions.
+
image::../images/Bucket_policy.png[Bucket_policy]

.. Click on `USERS` and add a new user with the name `CN=demonifi, O=Cluster Node`
+
image::../images/Add_users.png[Add_user]

... Click on `✏️` icon to edit the user. And add special previleges `Can manage buckets` and `Can proxy user requests`.
+
image::../images/Privileges.png[Privileges]

. Configuration on `NiFi` UI
.. Login to the `NiFi` UI.
.. Right click the canvas and select `Manage access policies` to open the `Access policies Settings`.
+
image::../images/Manage_access_policies.png[Manage_access_policies]

.. Create a new policy for `view the component` and Grant access to user `nifiadmin` and group `nifi-k8s-nodes`.
... Cick on `Create` button to create a new policy for `view the component`
... Add user `nifiadmin` and group `nifi-k8s-nodes` to the policy.
+
image::../images/Create_access_policy.png[Create_access_policy]
+
image::../images/Add_user_group.png[Add_user_group]
+
image::../images/view_component.png[view_component]

.. Similarly, create a new policy for `modify the component` and Grant access to user `nifiadmin` and group `nifi-k8s-nodes`.
+
image::../images/modify_component.png[modify_component]

.. Similarly, create a new policy for `operate the component` and Grant access to user `nifiadmin` and group `nifi-k8s-nodes`.
+
image::../images/operate_component.png[operate_component]
.. Similarly, create a new policy for `view the data` and Grant access to user `nifiadmin` and group `nifi-k8s-nodes`.
+
image::../images/view_data.png[view_data]

.. Add `NiFi Registry` as a registry client in the `NiFi` UI:
... Click on the three parallel lines in the upper right corner to open the `Controller Settings`.
+
image::../images/Controller_settings.png[Controller_settings]

... Select the `Registry Clients` tab.
... Click the `+` (Add Registry Client) button.
... Enter a name for the registry client (for example, `NiFi-Registry`).
... Provide the NiFi Registry URL (for example, https://demonifiregistry-web.demo-nifi-registry.svc.cluster.local:18443/nifi-registry[https://demonifiregistry-web.demo-nifi-registry.svc.cluster.local:18443/nifi-registry]).
... Optionally, add a description for the registry client.
... Click `Add` to save the registry client configuration.
... The NiFi Registry should now appear in the list of registry clients and be available for use in versioned flows.
+
image::../images/Add_registry_client.png[Add_registry_client]

. Run the demo flow
.. Drag `process group` to the canvas. Upload the `demo-flow.xml` file to the process group.
+
image::../images/Upload_demo-flow.png[Upload_demo-flow]
.. Right click the process group and select `version` -> `Start version control`.
+
image::../images/Start_version_control.png[Start_version_control]
.. Save flow version by providng a name and clicking on `Save` button.
+
image::../images/Save_flow_version.png[Save_flow_version]

.. You can now see the versioned flow in the `NiFi Registry` UI.
+
image::../images/Versioned_flow.png[Versioned_flow]

.. Right Click on the process group and select `Start` button to start the process group.
.. You can list the queued items with option `List Queues`.
.. Double Click on `AttributesToCSV` flow and check `terminate` option under `RELATIONSHIPS` to Terminate the flow.
+
image::../images/Terminate_flow.png[Terminate_flow]

== Streaming-Operators Demo(CSA and CSM)
=== Preparation
Before starting the demo, ensure you have the following prerequisites:

. `CSA Operator` with `Flink` and `CSM operator` with `Kafka` installed in your Kubernetes cluster. If not installed, follow the instructions in the link:./README.adoc[`**Installation Guide**`] to set it up.
. You're able to access the `Flink` and `SSB` web UIs. Follow the steps `4.3.` and `4.4.` in the link:./README.adoc[`**Installation Guide**`] to access UI.

. Download `kafka_shell.sh` from the Cloudera archive to manage kafka cluster. 
+
[source, bash]
----
# Update the [***cloudera_username***] and [***cloudera_password***] with your cloudera license paywall credentials.
export cloudera_username="[***cloudera_username***]"
export cloudera_password="[***cloudera_password***]"

# For Linux
curl -u "$cloudera_username:$cloudera_password" "https://archive.cloudera.com/p/csm-operator/1.3.0/tools/kafka_shell.sh" > kafka_shell.sh
chmod +x kafka_shell.sh

# For Mac - the additional sed command is only necessary until CSMDS-1175 is resolved
curl -u "$cloudera_username:$cloudera_password" "https://archive.cloudera.com/p/csm-operator/1.3.0/tools/kafka_shell.sh" | sed -E '/(Darwin|BASH_VERSINFO)/,/fi/s/^/#/g' > kafka_shell.sh
chmod +x kafka_shell.sh
----

.. You can now use `kafka_shell.sh` to run Kafka commands, such as `kafka-topics.sh`:
+
[source,bash]
----
#Initialize the kafka shell with the namespace and cluster name.
# Make sure to replace `cloudera-kafka-demo` with the namespace where your Kafka cluster is running and `kafka-cluster` with the name of your Kafka cluster.
./kafka_shell.sh --namespace=cloudera-kafka-demo --cluster="kafka-cluster"

#Once the above command is executed and prints out 'Shell ready', you will be in the kafka shell and can run kafka commands like below to list topics:
bin/kafka-topics.sh --list --bootstrap-server kafka-cluster-kafka-bootstrap:9092
----

=== Demo Steps

. Create Kafka topics named `demo-topic` and `demo-sql` in the cluster. These topics are required for the demo:
+
[source,bash]
----
./kafka_shell.sh --namespace=cloudera-kafka-demo --cluster="kafka-cluster"

# Once the above command is executed and prints out 'Shell ready', you will be in the kafka shell and can run kafka commands like below to create topics:
# Create  `demo-topic`
bin/kafka-topics.sh  --bootstrap-server kafka-cluster-kafka-bootstrap:9092 --topic demo-topic --create
# Create `demo-sql`
bin/kafka-topics.sh --bootstrap-server kafka-cluster-kafka-bootstrap:9092 --topic demo-sql --create
----

. Run a Kafka Data Generator Job in Flink
+
[source,bash]
----
kubectl apply -f demo-resources/demo-flinkjob.yaml -n csa-operator-system
----

.. Wait for the task manager pod to reach the `READY 1/1` state, then verify that data is being produced to the `demo-topic` topic with below command.
+
[source,bash]
----
# Check the status of the task manager pod
kubectl get pods -n csa-operator-system | grep taskmanager

# Login to kafka shell
./kafka_shell.sh --namespace=cloudera-kafka-demo --cluster="kafka-cluster"

#Once the above command is executed and prints out 'Shell ready', you will be in the kafka shell and can run kafka commands like below to consume messages from the topic:

bin/kafka-console-consumer.sh --topic demo-topic --bootstrap-server kafka-cluster-kafka-bootstrap:9092
# You should see messages being produced to the topic.
----
+
image::../images/Generated_data.png[Generated_data]

. Run a Flink SQL job to transform data from Kafka

.. login to the SSB UI and create a new Flink SQL job with the given link:./demo-resources/demo-flinksql.yaml[`**SQL query**`] :
... login to the SSB UI.
... Open project `ssb-default`
+
image::../images/Open_project.png[Open_project]

... Right click `jobs` and then `New Job`. Provide a name for the job.
+
image::../images/Create_job.png[Create_job]

... Paste the SQL query from the link:./demo-resources/demo-flinksql.yaml[`**SQL query**`] into the SQL editor.
+
image::../images/Paste_query.png[Paste_query]

... Select all and Click `Execute Selection` button to run the job. 
+
[NOTE]
====
It might throw timout error, again click `Execute Selection` button to run the job.
====

... Verify that the the data is being produced to the `demo-sql` topic with below command.
+
[source,bash]
----
./kafka_shell.sh --namespace=cloudera-kafka-demo --cluster="kafka-cluster"

#Once the above command is executed and prints out 'Shell ready', you will be in the kafka shell and can run kafka commands like below to consume messages from the topic:

bin/kafka-console-consumer.sh --topic demo-sql --bootstrap-server kafka-cluster-kafka-bootstrap:9092
# You should see transformed messages being produced to the topic.
----
+
image::../images/Transformed_data.png[Transformed_data]
